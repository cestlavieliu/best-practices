对当前分支相对于主分支的所有变更进行代码审查。

## 步骤

1. 确定主分支名称（main 或 master）
2. 运行 `git diff <主分支>...HEAD` 获取所有变更
3. 如果没有 diff（比如还没有新提交），则回退到 `git diff` 查看未提交的变更
4. 逐文件分析变更内容

## 审查重点

### Bug 和逻辑问题
- 逻辑错误、边界条件遗漏
- 空指针/未定义引用风险
- 错误处理是否完整
- 竞态条件、死锁风险

### 安全漏洞
- XSS、SQL 注入、命令注入
- 敏感信息硬编码（密钥、密码、token）
- 不安全的反序列化
- 权限检查缺失

### 代码风格和质量
- 命名是否清晰表达意图
- 函数/方法是否过长或职责过多
- 重复代码
- 不必要的复杂度

## 输出格式

用中文输出审查结果，按以下格式组织：

### 📋 变更概览
简要说明本次变更的目的和范围。

### 🔴 必须修复
严重问题（Bug、安全漏洞），每项包含：
- 文件路径和行号
- 问题描述
- 修复建议

### 🟡 建议改进
非阻塞性建议，每项包含：
- 文件路径和行号
- 当前问题
- 改进方案

### ✅ 做得好的地方
值得肯定的实践（简要列出）。

### 总结
一句话总结审查结论：可以合并 / 需要修改后再合并 / 需要重大修改。
